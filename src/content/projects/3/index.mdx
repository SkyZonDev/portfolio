### Le Problème

Les solutions d'authentification actuelles nous forcent à choisir entre :

- **Services tiers** (Auth0, Firebase, Cognito) avec leurs dépendances externes et leur verrouillage
- **Protocoles complexes** (OAuth 2.0, OIDC) surdimensionnés pour de simples besoins backend

### La Solution

AIS Forge adopte une approche différente : un serveur headless, auto-hébergé, qui implémente uniquement ce dont vous avez vraiment besoin.

## Caractéristiques Principales

### Sécurité Sans Compromis

- Tokens JWT avec signature asymétrique
- Rotation automatique des clés cryptographiques
- Tokens de rafraîchissement opaques avec rotation stricte
- Aucun secret stocké en clair
- Journalisation complète des audits

### Architecture Moderne

Construit avec les meilleures technologies pour la performance et la fiabilité :

- **Fastify** : Framework Node.js haute performance
- **PostgreSQL** : Base de données robuste et éprouvée
- **Drizzle ORM** : Type-safety de bout en bout
- **TypeScript** : Sécurité au niveau du code

### API Simple et Puissante

```typescript
// Authentification en 3 lignes
const { access_token, refresh_token } = await fetch('/api/v1/auth/sign-in', {
  method: 'POST',
  body: JSON.stringify({ email, password })
}).then(r => r.json());

// Utilisation immédiate
fetch('/api/resource') // Via les cookies

fetch('/api/resource', {
  headers: { 'Authorization': `Bearer ${access_token}` }
}); // Via un bearer token
```

## Cas d'Usage Idéaux

Ce projet est parfait pour :

- Plateformes internes d'entreprise
- Backends SaaS auto-hébergés
- Architectures microservices
- Applications nécessitant une authentification centralisée
- Environnements où la conformité impose l'auto-hébergement

## Architecture Technique

```
Backend App → AIS Forge Auth Server → PostgreSQL
              ├─ Gestion des identités
              ├─ Émission de tokens JWT
              ├─ Rotation automatique des clés
              └─ Endpoint JWKS public
```

Le serveur expose une API REST simple, sans interface utilisateur, conçue pour être interrogée par vos applications backend.

## Philosophie du Projet

**Pas de bloat, que l'essentiel**

AIS Forge refuse la complexité inutile. Pas de redirections navigateur, pas de protocoles sur-ingéniérisés. Juste une authentification solide, moderne et contrôlable.

**Sécurité par conception**

Chaque décision technique priorise la sécurité : cryptographie asymétrique, tokens à courte durée de vie, rotation obligatoire, audit complet.

**Votre infrastructure, vos règles**

Auto-hébergé signifie contrôle total : pas de quotas, pas de tarification surprise, pas de changements d'API imposés par un tiers.
